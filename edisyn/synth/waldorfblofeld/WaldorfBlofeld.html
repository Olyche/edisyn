Waldorf Blofeld

Single Patch Editor

This editor will work with the Waldorf Blofeld, the Waldorf Blofeld with SL Option, and the Waldorf Blofeld Keyboard (which automatically comes with the SL option).

Communicating with Edisyn Set your Blofeld to send Sysex only, not Ctl nor Ctl+Sysex. Also make sure that Ctrl Receive is on.

SL Option The SL option gives you the option of specifying a sound sample instead of a wave on Oscillators 1 and 2. This is turned on by choosing a Sample Bank in the editor: then the wave menu turns into a sample menu. You turn it off by choosing None in the Sample Bank. BTW, if you're wondering why Edisyn's SL Sample Banks end at Bank L, that's the furthest a contributor could get his to go before crashing his Blofeld. So there you go.

Uploading Wavetables Under the Blofeld menu you'll find an option to upload a wavetable. You can upload wavetables from three different formats:

  • WAV files from http://waveeditonline.com This is a large collection of WAV files encoding wavetables for other devices. You can even download all of them as one zip file. Edisyn will convert them to the Blofeld as best it can (resampling to the Blofeld's lower sampling rate). These are particularly good because there is a very nice free wavetable editor available, http://synthtech.com/waveedit

  • Sysex files in SYX or MID format. These are groups of one or more wavetables in standard Blofeld sysex. Some good sources for these are:
    • Stromeko's Waldorf downloads page. http://synth.stromeko.net/Downloads.html

    • MarsMelons's custom wavetables. http://www.marsmelons.com/custom-wavetables-waldorf-blofeld/

    • Big Gnome's VFX wavetables. https://tinyurl.com/vfxwavetables (from https://waldorfmusic.com/en/forum/blofeld-en/76-sharing-custom-set-of-blofeld-wavetables)
Important note: You probaby want to turn Limit WT off when using user wavetables. Otherwise the Blofeld will not work properly with any PWM values greater than 120.

Uploading Pseudo-MPE The Blofeld doesn't support MPE. But it does support 16 separate patch slots in a multimode patch, which can be taken advantage of by instruments such as the Linnstrument. To do this each slot must be identical except for a unique MIDI channel, and each slot must be set to the desired single patch. You can do this in two ways.

  • Write Patch as Psuedo-MPE    This writes a multimode patch which points to your current single patch, including the current name, bank, and (single patch) number. You get to choose what multimode patch number to write to.

  • Write Pseudo-MPE on Batch Download    If you check this, then if you start a batch download, for each downloaded patch, Edisyn will automatically write a multimode patch which points to it, including the current name, bank, and (single patch) number. The multimode patch number will be the same as the single number. Thus if you download exactly one bank of single patches, you'll have multimode patches loaded for all of them.

    Why is this connected to batch download? Because in order to properly write multimode patches, Edisyn needs to know the name of the single-mode patch to which they refer (so it can call the multimode patch by the same name). To do this it must load the single-mode patch. The easiest way to load single-mode patches in bulk is by hijacking the bulk download facility. So there you have it.

Gotchas Only two at present:

  • If you load the current patch, the Blofeld does not tell Edisyn what bank or number the patch is.

  • If you send a patch, or modify its parameters, the Blofeld's patch screen does not indicate that it's been edited (as it usually does with a little inverted "E"), even though it has in fact been edited.
